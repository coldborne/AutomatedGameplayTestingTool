# Требования

## 1. Цель
Создать небольшую игровую сцену, на которой можно проверить базовые игровые механики с помощью автоматизированного бота.

## 2. Функциональные требования
### 2.1 Перемещение бота
Бот должен перемещаться по заранее заданному маршруту от стартовой точки до финиша.

### 2.2 Взаимодействие с кнопкой
Кнопка должна изменить состояние на `pressed` при входе бота в зону кнопки .

### 2.3 Открытие двери
После нажатия кнопки связанная дверь должна открыться.

### 2.4 Захват точки
Прогресс захвата должен увеличиваться до 100% при попадании бота в зону захвата.

### 2.5 Завершение маршрута
После прохождения всех контрольных точек бот должен завершить сценарий.

## 3. Нефункциональные требования
### 3.1 FPS
Средний FPS во время тестового сценария не должен быть ниже 30.

### 3.2 Система опознования застревания
Бот не должен находиться без движения дольше 2 секунд, если сценарий ещё не завершён. В ином случае система определяет бота как застрявший unit.

### 3.3 Отчётность
После завершения сценария должен формироваться JSON-отчёт с результатами проверок.

## 4. Критерии приёмки
Система считается корректной при выполнении следующих условий:
- бот успешно выполняет поставленные задачи
- При успешном или неудачном выполнении сценария, формируется отчёт с подробным состоянием системы для понимания на каком этапе произошла ошибка
